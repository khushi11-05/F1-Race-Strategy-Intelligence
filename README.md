## 📌 Project Summary

This project ingests real F1 timing and telemetry data via the FastF1 API, performs
statistical analysis on race strategy patterns, trains a machine learning classifier
to predict podium finishes, and presents all findings through a live Streamlit dashboard.

| Metric | Value |
|---|---|
| Season covered | 2023 (Rounds 1–3) |
| Races | Bahrain · Saudi Arabia · Australia |
| Drivers | 20 |
| Teams | 10 |
| Total lap records | 3,002 |
| Clean racing laps (post-filter) | 2,469 |
| ML model accuracy (LOO-CV) | 90.0% |
| Podium recall | 88.9% |
| Podium F1-score | 0.727 |

## 🔧 Setup & Usage

### 1. Install dependencies
```bash
pip install fastf1 pandas numpy matplotlib seaborn scipy scikit-learn streamlit plotly
```

### 2. Collect data (Phase 1)
```bash
python f1_data_collection.py
```
This creates the `f1_data/` folder with 4 CSVs. Set `TARGET_YEARS` and `MAX_ROUNDS`
in the script to control how much data to pull. The `f1_cache/` folder speeds up
subsequent runs by caching API responses locally.

### 3. Run EDA notebook (Phase 2)
```bash
jupyter notebook f1_phase2_eda.ipynb
```

### 4. Run ML notebook (Phase 3)
```bash
jupyter notebook f1_phase3_ml.ipynb
```

### 5. Launch dashboard (Phase 4)
```bash
streamlit run app.py
```
Opens at `http://localhost:8501`

## 📊 Phase 2 — Key Findings

### Qualifying → finishing position
- **Spearman r = 0.685** (p < 0.0001) between grid and finishing position
- **83% of P1–P5 starters** retained a top-5 finish
- Correlation is strong but not deterministic — strategy and DNFs create movement

### Tire compound analysis
| Compound | Median lap time | Typical stint length |
|---|---|---|
| HARD | 94.0s | 15–26 laps |
| MEDIUM | 95.3s | 8–15 laps |
| SOFT | 98.6s | 5–12 laps |

Counter-intuitive finding: **HARD tires produce the fastest median lap times** overall.
This is because HARD compounds are run in long, settled stints where track rubber
builds up — not because they are intrinsically faster over a single lap.

A **Kruskal-Wallis test** (H = 543.7, p ≈ 10⁻¹¹⁹) confirmed the compound
lap time differences are statistically significant.

### Pit stop strategy
- Most common strategy: **2-stop**
- Australian GP was a high-attrition round (red flag), inflating average stops to 3.6
- Bahrain and Saudi Arabia averaged 2.6 and 1.25 stops respectively

### DNF rate
- **21.7%** of race entries did not finish (across 3 rounds)
- Australia significantly inflated this figure due to the red flag incident

## 🤖 Phase 3 — Machine Learning

### Problem definition
**Binary classification:** will a driver finish on the podium (P1/P2/P3)?

### Features engineered

| Feature | Source | Rationale |
|---|---|---|
| `GridPosition` | Results | Qualifying performance |
| `GridTop5` | Derived | Binary flag — qualified in top 5 |
| `NumStops` | Pit data | Strategy aggressiveness |
| `TrackTemp` | Weather | Affects tire behaviour |
| `MaxTyreLife` | Lap data | Proxy for stint length / undercut |
| `TeamRank` | Derived | Constructor performance ranking |
| `ConstructorPts` | Results | Car quality proxy |

### Model comparison (Leave-One-Out CV)

| Model | Accuracy | Precision | Recall | F1 |
|---|---|---|---|---|
| **Logistic Regression** ✓ | **0.900** | **0.615** | **0.889** | **0.727** |
| Random Forest | 0.883 | 0.583 | 0.778 | 0.667 |
| Gradient Boosting | 0.867 | 0.545 | 0.667 | 0.600 |

**Why Logistic Regression wins here:** with only 60 training samples, simpler models
generalise better. Logistic Regression outperforms tree ensembles on F1 score — a
good reminder that model complexity should match data size.

**Why LOO-CV:** with 60 rows, standard k-fold wastes too much training data. Leave-One-Out
trains on 59 samples and tests on 1 each time, giving the most honest estimate possible.

### Feature importance (permutation, 30 repeats)

| Rank | Feature | Importance |
|---|---|---|
| 1 | TeamRank | +0.104 ± 0.079 |
| 2 | ConstructorPts | +0.097 ± 0.075 |
| 3 | MaxTyreLife | +0.053 ± 0.041 |
| 4 | NumStops | +0.040 ± 0.032 |
| 5 | GridTop5 | +0.039 ± 0.048 |

**Key insight:** car quality (`ConstructorPts`, `TeamRank`) accounts for more predictive
power than qualifying position. In F1 at this sample size, having the best car is
a stronger predictor of podium than where you start.

### Known limitations
- 60 samples is a small dataset — scale to full seasons (460+ entries/season) for robust results
- `NumStops` and `MaxTyreLife` are post-race features; a true pre-race predictor would use
  planned strategy data instead
- Random events (safety cars, mechanical failures) introduce irreducible noise

## 🚀 How to Extend This Project

1. **More seasons** — set `TARGET_YEARS = range(2018, 2025)` in Phase 1 to get 7 seasons
2. **Add qualifying lap time** as a feature — more precise than grid position
3. **Rolling driver form** — avg points in last 3 races as a feature
4. **XGBoost + GridSearchCV** — tune hyperparameters with more data
5. **Shift target to top-5** — gives more positive samples for better class balance
6. **Deploy on Streamlit Cloud** — free hosting with a shareable public URL
